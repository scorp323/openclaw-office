# Spec: 工作流编辑器 UX 改进

## 场景一：Args JSON 新增字段

**前提**：用户正在编辑一个 `openclaw-invoke-step` 节点，节点的 Args JSON 当前为空（`undefined`）或有若干键值对。

**操作**：用户点击「+ Add Field」按钮。

**期望结果**：
- 立即在键值对列表下方出现一行新的空白输入（key 默认为 `new_key`，value 为空字符串）
- 用户可以立即在新行中输入 key 和 value
- 修改完成后，数据正确序列化回 `step.command`

**验收标准**：
- [ ] value 为 `undefined` 时点击 Add 有效
- [ ] value 为已有对象时点击 Add 有效（追加而非覆盖）
- [ ] 新增字段后可正常删除（× 按钮有效）
- [ ] key 重名时后者覆盖前者（与原逻辑一致）

---

## 场景二：Tool Name 选择器

**前提**：用户正在编辑一个 `openclaw-invoke-step` 节点。

**操作**：用户查看/修改 Tool Name 字段。

**期望结果**：
- 字段显示为下拉选择器，列出当前 Gateway 可用工具
- 下拉加载时显示 loading 状态
- 加载失败时回退为自由输入框
- 选择某个工具后，Action 字段自动更新为该工具支持的操作列表

**验收标准**：
- [ ] 下拉显示 Gateway tools.catalog 返回的工具列表
- [ ] 可以选择工具并正确写入 command
- [ ] Action 下拉根据 Tool 动态变化
- [ ] 离线/未连接 Gateway 时字段不崩溃，回退到 input

---

## 场景三：拖入新节点自动插入到已有链路

**前提**：画布上存在 A → B 的连线。

**操作**：用户从调色板拖一个新节点 C，释放到 A→B 连线附近（距离 < 30px）。

**期望结果**：
- 原 A→B 连线消失
- 新增 A→C 和 C→B 两条连线
- C 节点位于释放位置

**验收标准**：
- [ ] 释放到边上时，原边被删除，新增两条边
- [ ] 释放到空白区域时，行为与原来相同（只添加节点，不影响任何边）
- [ ] 插入后 undo 可以回到插入前状态

---

## 场景四：STDIN 步骤引用选择器

**前提**：用户正在编辑一个节点，需要设置 STDIN 引用上游步骤的输出。

**操作**：用户点击 STDIN 字段。

**期望结果**：
- 字段显示当前工作流中所有其他步骤 ID 的输出引用建议（`$stepId.stdout`）
- 用户可以从建议中选择或手动输入自定义值

**验收标准**：
- [ ] 建议列表包含当前所有步骤（排除当前编辑的步骤）
- [ ] 选择建议后文本正确填入
- [ ] 手动输入任意值仍然有效
