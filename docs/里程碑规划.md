# OpenClaw Office — 分阶段里程碑规划

> 版本: v1.0 | 日期: 2026-02-25
> 总工期估算: 6 周（含缓冲），分 3 个 Phase 交付

---

## 总览

```
Phase 1: 基础框架 + 2D 平面图          (Week 1-2)   ← 最小可用产品
Phase 2: Isometric 2.5D 办公室         (Week 3-4)   ← 核心体验
Phase 3: 3D 增强 + 高级交互            (Week 5-6)   ← 完整愿景
```

每个 Phase 结束后都有一个可交付、可演示的产品版本。

---

## Phase 1: 基础框架 + 2D 平面图（Week 1-2）

**目标：** 打通 Gateway → 前端的完整数据链路，用 2D SVG 平面图展示 Agent 实时状态。

### Milestone 1.1 — 项目脚手架（Day 1-2）

| 任务 | 交付物 | 验收标准 |
|------|--------|---------|
| 初始化 Vite + React + TypeScript 项目 | `apps/office/` 目录结构 | `pnpm dev` 可启动开发服务器 |
| 配置 Tailwind CSS 4.x | 全局样式 | Tailwind 类名生效 |
| 配置 Zustand store 基础骨架 | `office-store.ts` | store 可创建、读写 |
| 创建 AppShell 布局组件 | 顶栏 + 侧栏 + 主区域 | 响应式布局正常 |
| 配置 pnpm workspace | root `pnpm-workspace.yaml` 引入 | `pnpm install` 正常 |

**开发指导（给 Codex）：**
```
1. 在 apps/office/ 下用 Vite 创建 React + TypeScript 项目
2. pnpm-workspace.yaml 添加 "apps/*"
3. Tailwind 4.x 使用 @tailwindcss/vite 插件
4. Zustand 5.x 使用 create() + 中间件 immer
5. AppShell 布局参考设计图4（左侧3D/2D + 右侧面板，可折叠）
```

### Milestone 1.2 — Gateway WebSocket 客户端（Day 2-3）

| 任务 | 交付物 | 验收标准 |
|------|--------|---------|
| 实现 WS 连接 + challenge 认证 | `gateway/ws-client.ts` | 连接 Gateway 后收到 `connect.accepted` |
| 实现自动重连（指数退避） | 重连逻辑 | 断开后 1-30s 内自动重连 |
| 实现 RPC 请求封装 | `gateway/rpc-client.ts` | 可调用 `agents.list` 并获得响应 |
| 实现事件解析器 | `gateway/event-parser.ts` | 正确解析 `{ type: "event", event: "agent", payload }` |
| 连接状态 UI 指示器 | TopBar 组件 | 显示 connected/disconnected/reconnecting |

**开发指导：**
```
1. 查阅 src/gateway/protocol/schema/frames.ts 了解 Gateway 帧格式
2. connect 请求参考 src/gateway/server/ws-connection.ts 中的认证流程
3. client.mode 使用 "companion"，caps 包含 "tool-events"
4. RPC 使用 { type: "req", id: <uuid>, method, params } 格式
5. 重连策略：Math.min(1000 * 2^attempt, 30000) + 随机抖动
```

## Phase 2: Isometric 2.5D 办公室（Week 3-4）

**目标：** 将 2D 平面图升级为 isometric 视角的 2.5D 办公室场景，增加 Sub-Agent 生命周期可视化和会议区交互。

### Milestone 2.1 — R3F 基础场景（Day 8-10）

| 任务 | 交付物 | 验收标准 |
|------|--------|---------|
| 集成 React Three Fiber | `Scene.tsx` | R3F Canvas 正常渲染 |
| Isometric 相机 | OrthographicCamera 配置 | 45度等距投影视角 |
| 环境搭建 | `Environment.tsx` | 灯光 + 地板 + 网格线 |
| 简单工位模型 | BoxGeometry 代替 | 桌子 = 长方体，椅子 = 小方块 |
| 区域划分 | 3D 地面标记 | 工位区/会议区/热工位区 有色彩区分 |
| 2D/3D 模式切换 | UI 按钮 | 可在 2D SVG 和 3D R3F 间切换 |

**开发指导：**
```
1. OrthographicCamera 设置：
   - position: [10, 10, 10]
   - zoom: 50
   - near: 0.1, far: 1000
2. 地板用 PlaneGeometry 旋转 -π/2，材质半透明网格
3. 桌椅用 BoxGeometry，后续替换为 GLTF
4. drei 的 <Grid>, <Environment>, <OrbitControls> 可直接使用
5. 使用 drei 的 <Html> 组件将气泡/面板嵌入 3D 场景
```

### Milestone 2.2 — Agent 角色系统（Day 10-12）

| 任务 | 交付物 | 验收标准 |
|------|--------|---------|
| Agent 3D 角色 | `AgentCharacter.tsx` | 简单几何体小人（胶囊体 + 球头） |
| 确定性 Avatar 生成 | `avatar-generator.ts` | 基于 agentId hash 生成颜色/形状 |
| 状态动画 | 动画状态机 | idle=微晃、thinking=头顶旋转圈、tool=面前浮板、speaking=气泡 |
| Sub-Agent 外观 | 半透明材质 | Sub-Agent 使用半透明 + 闪烁效果，区分于常驻 Agent |
| 工位分配 | `position-allocator.ts` | 常驻 Agent 固定位、Sub-Agent 动态分配 |

**开发指导：**
```
1. 角色由 CapsuleGeometry（身体）+ SphereGeometry（头）组成
2. 颜色用 MeshStandardMaterial，颜色由 avatar-generator 提供
3. idle 动画：useFrame 中做 sin 波小幅 Y 轴位移
4. thinking 动画：头顶 Torus 旋转
5. Sub-Agent 用 transparent + opacity: 0.6 + 蓝色调
6. 移动使用 framer-motion-3d 或 useSpring 做位置过渡
```

### Milestone 2.3 — Sub-Agent 生命周期可视化（Day 12-13）

| 任务 | 交付物 | 验收标准 |
|------|--------|---------|
| Sub-Agent 生成动画 | `SpawnPortal.tsx` | 父 Agent 位置出现传送门 + 子 Agent 在新位置闪现 |
| Sub-Agent 消失动画 | 淡出 + 缩小效果 | endedReason 不同有不同动画（complete=绿色消散、error=红色碎裂） |
| 父子连线 | 虚线光束 | 父 Agent 和子 Agent 之间的连线 |
| Sub-Agent 列表面板 | 面板 UI | 显示当前所有 Sub-Agent 及其状态、parent、task |

**开发指导：**
```
1. 订阅 subagent_spawned / subagent_ended plugin hook 事件
   （需确认 Gateway 是否已将这些事件通过 WS 广播）
2. 传送门特效：3 个同心 Ring 旋转 + 发光（drei 的 <Sparkles> 或自定义 shader）
3. 消失动画：scale 从 1→0 + opacity 从 1→0，0.8s 过渡
4. 父子连线使用 drei 的 <Line> 组件，虚线 dashScale
```

## 风险登记

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| Gateway WS 认证流程不兼容 | 中 | Phase 1 阻塞 | Week 1 第一天就跑通连接 |
| 3D 模型获取/制作耗时 | 高 | Phase 3 延期 | Phase 2 坚持用几何体，Phase 3 寻找免费低多边形素材 |
| Sub-Agent 事件未通过 WS 广播 | 中 | Phase 2 功能缺失 | 降级为 RPC 轮询 sessions.list |
| 大量 Agent 并发时 3D 性能不足 | 中 | 用户体验差 | 超过阈值自动切 2D，用 InstancedMesh |
| Codex 实现偏离设计 | 中 | 返工 | 每个 Milestone 结束做 Review |

## Codex 执行指导

### 开发顺序
严格按 Phase 1 → 2 → 3 的 Milestone 顺序执行。每个 Milestone 完成后确认验收标准全部通过再进入下一个。

### 代码规范
- TypeScript strict 模式，不用 `any`
- 文件不超过 500 行，超过则拆分
- 组件命名 PascalCase，hook 命名 useCamelCase
- 遵循 OpenClaw 主项目的 Oxlint + Oxfmt 规范

### 测试要求
- Store 和 event-parser 必须有单元测试
- 组件使用 vitest + @testing-library/react 测试关键交互
- 不要求高覆盖率，但关键数据流必须有测试

### 关键参考文件（在主项目中）
- `src/infra/agent-events.ts` — Agent 事件类型定义
- `src/gateway/protocol/schema/frames.ts` — WS 帧格式
- `src/gateway/server/ws-connection.ts` — WS 认证流程
- `src/agents/subagent-spawn.ts` — SubAgent 类型
- `src/agents/subagent-registry.types.ts` — SubAgent 运行记录
- `src/agents/subagent-lifecycle-events.ts` — SubAgent 生命周期状态
- `src/gateway/server-methods-list.ts` — 所有 Gateway 事件名
- `src/config/types.agents.ts` — Agent 配置类型
- `src/plugins/types.ts` — Plugin API 类型（如需做成插件）

### Mock 数据策略
在 Gateway 不可用时，应有 Mock 模式：
- `apps/office/src/gateway/mock-provider.ts` — 模拟 WS 事件流
- 每 2-5 秒随机生成 lifecycle/tool/assistant 事件
- 支持 `VITE_MOCK=true` 环境变量启用
